Is any frontier model ready for regulated finance?

A regulation-anchored, calibration-validated, fully reproducible scorecard of model behavior — not a system audit, not a certification. Ten models, two deployment profiles, graded against preregistered bars drawn from the EU AI Act, FINMA, DORA, revFADP and banking-secrecy law, with judgment cases scored by a cross-family LLM panel calibrated to a human rater.

The scorecard

Each cell is one requirement for one model, on one deployment profile. The verdict is absolute — measured against a preregistered floor/target, never a curve. A single Critical failure means Not-ready: it cannot be averaged away. Click any cell to read the actual scenarios and the model's own responses behind the verdict.

Per-requirement detail

Pass-rate per model for the selected requirement, with the preregistered floor and target drawn. Above target = pass; between = partial; below floor = fail.

Why trust the grader?

The judgment cases are graded by a cross-family LLM panel — but only because it was first calibrated against a human rater on a 150-item blind pack. Two independent humans agree with each other far more than either agrees with the old rule grader, and the panel agrees with the human better than any single rule. Nothing is graded by a layer that didn't clear a preregistered bar.

Per-family deployment gate

Krippendorff's α of the panel vs the human rater, per judgment family, against the preregistered bands. α is prevalence-fragile where one class is rare — read raw agreement + fail-recall alongside.

Regulation cross-walk

Every requirement is anchored to named regulation, and gates differently by deployment profile (an anonymous public bot must refuse all personal data; an authenticated assistant may serve the logged-in user but never another).

How it works

Regulation-anchored

Each requirement carries an industry-specific regulatory anchor; thresholds are derived from the law, not from a leaderboard curve.

Preregistered

Floor/target bars are fixed before a model is run, never tuned to a result; each card stamps its thresholds, dataset version and run date.

Reproducible

Every verdict re-derives from a frozen, committed transcript at zero model cost (make repro) — judge majorities re-derive from cached per-judge votes, offline.

Honest

No single 0–100 score. A readiness verdict is a structured signal, not regulatory approval; a ✗ sizes the system wrap a deployer must build, it is not "a bad model".

Model-behavior cards — not a conformity assessment, not legal advice. A green result is model-layer only and does not certify a deployed system. Scenarios are synthetic.
